Frontend Endpoint Map

This map is based only on the React frontend code for EquipoBraulio, EquipoCristobalRios, and EquipoGerman. It lists the endpoints and browser media requests the frontend performs for the target performance actions.

Audit Boundary

No backend files were used for this version. Response status notes and rate-limit details were removed because this document is now a frontend-source-of-truth map. Media rows come from rendered <img> and <video> elements whose URLs are built by the frontend from API response fields.

Request Headers

JSON reads use the browser default fetch headers unless a row says otherwise. Multipart writes are FormData; do not manually set the boundary. JSON writes set Content-Type: application/json. EquipoGerman protected writes add Authorization: Bearer $TOKEN after POST /auth/token.

Not Counted In The Target Actions

The frontends also contain subscription feed, subscribe/unsubscribe, and delete-video flows. They are real frontend endpoints, but they are not part of the target action set documented above unless a performance scenario explicitly includes those clicks.

Examples: Braulio and Cristobal call GET /users/{userId}/feed, POST/DELETE /users/{userId}/subscriptions/{creatorId}, and DELETE /videos/{videoId}?requester_user_id={id}. German calls GET /api/users/{userId}/feed?limit=20&offset={n}, POST/DELETE /api/users/{userId}/subscriptions/{creatorId}, and DELETE /api/videos/{videoId}.
