Header rules

Accept: application/json is enough for JSON reads. JSON writes need Content-Type: application/json. Multipart writes need Content-Type: multipart/form-data; in K6, let the runtime generate the boundary. Browser image/video requests can send image/video Accept headers, and video playback may send Range: bytes=0-1048575. German's protected writes also need Authorization: Bearer $TOKEN.

Recommended action mix

Main page 34%, user page 5%, create user 1%, upload video 1%, open/watch/reproduce video 48%, scroll/load more 6%, add comment 5%.

Rate-limit scope

EquipoBraulio: no app-level rate limiter was found in the backend code. EquipoCristobalRios: SlowAPI uses the request client IP as the key, so all K6 virtual users from one EC2 instance share the same rate-limit bucket even if they use different user ids in payloads. EquipoGerman: a Redis limiter exists and keys by client IP + route + scope, not JWT user, but the inspected routes do not currently attach the limiter dependencies; if enabled later, different JWTs from the same EC2 instance would still share each route bucket.

K6 notes

Tag every K6 request with project, action, and a short endpoint name. For grouped browser actions, issue the endpoint rows in the same order/parallelism the frontend uses where practical. Use GET /health before each measured run, but keep health checks outside the action distribution.
